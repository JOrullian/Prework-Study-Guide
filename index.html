HTML

  • HTML elements work together to build a framework for a webpage.
  • Elements are usually of an opening and closing tag represented with angle brackets < >.
  • Meta elements are usually not seen by the end user, one exception is the "title" element.
  • The "head" element contains information about the webpage.
  • The "body" element represents the visible content shown to the user
  • HTML elements are case sensitive.

CSS

  • A CSS rule is divided into a selector and a declaration block
  • The selector defines the element or attribute to which the rules will apply
  • The declaration contains the CSS property to be applied and the value of the property
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • "Do Not Repeat Yourself" is a coding principle that reduces the overall complexity of a codebase

Git

  • git init: initializes local directory as new git repository.
  • git status: checks what branch we are currently on.
  • git diff: shows you the changes in your unstaged code.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add file-path/name-of-file: takes all unstaged work from a specific file and stages it, making it ready to be committed.
  • git commit -m "write commit message here": commits all staged work. It's important to write a brief message on what is being committed.
  • git pull: pulls changes from your remote space.
  • git push: pushes changes to your remote space.
